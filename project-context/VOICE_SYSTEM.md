VOICE SYSTEM CONTEXT

This project is a voice-first application.

Voice interaction is the primary interface.

Text and visual UI are secondary.

---

VOICE PIPELINE

audio capture  
→ speech-to-text  
→ AI reasoning  
→ text-to-speech  
→ audio playback

The system should support continuous conversational interaction.

---

VOICE CONSTRAINTS

Voice systems must consider:

microphone permissions  
audio autoplay restrictions  
speech recognition reliability  
network latency  
audio playback delays

These constraints are especially important in mobile browsers.

---

BROWSER ENVIRONMENT

The system must work reliably on:

iPhone Safari  
Android Chrome  
Samsung Internet  
desktop browsers

Different browsers may behave differently when handling audio.

---

PERFORMANCE GOAL

The system should minimize the delay between:

user speech  
and  
AI voice response

The conversation should feel natural and responsive.

---

USER EXPERIENCE GOAL

The interaction should feel like speaking with a calm AI conversation partner.

The system should encourage users to keep speaking.
