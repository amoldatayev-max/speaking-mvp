ENGINEERING PRINCIPLES

Prefer:

simple solutions  
clear code structure  
predictable system behavior  
low latency

Avoid:

overengineering  
complex frameworks  
unnecessary abstractions

---

CODE STYLE

Code should be:

clear  
maintainable  
well structured

Prefer small and focused modules.

---

PERFORMANCE

Voice applications are sensitive to latency.

Avoid:

blocking operations  
unnecessary network requests  
large payloads

Prefer asynchronous and efficient operations.

---

ERROR HANDLING

Always consider possible failures such as:

microphone permission denied  
speech recognition failure  
network errors  
API request failures  
audio playback issues

Code should handle these cases safely.

---

MOBILE STABILITY

The system must work reliably on:

iPhone Safari  
Android Chrome  
desktop browsers
